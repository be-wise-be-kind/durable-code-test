Observability Architecture

4-pillar observability with Grafana stack: metrics, logs, traces, and profiling with cross-pillar correlation

Four Pillars Overview

The observability stack covers four complementary signal types, each handled by a dedicated Grafana ecosystem component running in single-binary mode on a shared EC2 instance.

Metrics Mimir

Port: 9009

Memory: 768 MB

Long-term Prometheus-compatible metrics storage. Handles the 4 Golden Signals (latency, traffic, errors, saturation) and custom application metrics.

Logs Loki

Port: 3100

Memory: 384 MB

Log aggregation with label-based indexing. Stores structured JSON logs with trace_id and span_id fields for correlation.

Traces Tempo

Port: 3200

Memory: 384 MB

Distributed trace storage. Captures full request lifecycle from browser through backend with latency waterfall visualization.

Profiling Pyroscope

Port: 4040

Memory: 384 MB

Continuous profiling with flame graphs. Links CPU/memory profiles to specific traces and spans for root cause analysis.

System Architecture

Data flows from application sources through Alloy (unified collector) to backend storage, with Grafana providing visualization and correlation.

Resources are split across base (persistent) and runtime (ephemeral) workspaces to preserve data across infrastructure lifecycle operations.

Component Details

Each Grafana stack component runs in single-binary mode within Docker Compose on a shared t3.medium EC2 instance (4 GB RAM, ~2.7 GB allocated to containers).

Visualization Grafana

Memory: 384 MB

Port: 3001

Mode: Standalone

Unified dashboard and alerting UI. Reads from all 4 backend datasources. Serves from /grafana/ subpath via ALB.

Metrics Mimir

Memory: 768 MB

Port: 9009

Mode: Single-binary

Prometheus-compatible long-term metrics storage. Handles exemplars for trace correlation. S3 block storage.

Logs Loki

Memory: 384 MB

Port: 3100

Mode: Single-binary

Label-based log aggregation. TSDB index with S3 chunk storage. Structured JSON with trace_id/span_id fields.

Traces Tempo

Memory: 384 MB

Port: 3200

Mode: Single-binary

Distributed trace backend. Receives OTLP from Alloy, generates span metrics, enables trace-to-logs/profiles navigation.

Profiling Pyroscope

Memory: 384 MB

Port: 4040

Mode: Single-binary

Continuous profiling with flame graphs. Push-based SDK integration. Links CPU/memory profiles to traces via span labels.

Collector Alloy

Memory: 384 MB

Ports: 4317 (gRPC), 4318 (HTTP), 12347 (Faro)

Mode: Single-binary

Unified telemetry collector. Receives OTLP from backend, Faro from frontend, and forwards to all storage backends.

Network & Security

The observability EC2 instance runs in a public subnet (for Docker image pulling without NAT gateway cost). All inbound traffic is restricted by security group to ALB and ECS task sources. Grafana has no ALB listener rule — it is accessible only via SSM port forwarding.

Port Protocol Source Purpose
12347 HTTP ALB Security Group Faro telemetry collection via /collect/* path (only observability endpoint on public ALB)
4317 gRPC ECS Security Group OTLP telemetry from backend (gRPC)
4318 HTTP ECS Security Group OTLP telemetry from backend (HTTP)
4040 HTTP ECS Security Group Pyroscope push endpoint for profiling data
3001 HTTP SSM tunnel only (localhost) Grafana UI — no ALB route, no SG ingress from ALB

Security Posture

  • Public subnet, private access: EC2 has a public IP for outbound Docker pulls; all inbound restricted by security group
  • Grafana isolation: No ALB listener rule, no security group ingress from ALB. Accessible only via SSM port forwarding (requires AWS IAM credentials)
  • /collect/* is the sole public endpoint: Protected by WAF rate limiting (2000 req/5min/IP), OWASP Core Rules, and CORS origin restriction
  • S3 bucket policy: Blocked public access, instance profile grants write permission, AES-256 encryption at rest
  • Non-root containers: Storage backends run as UID 10001; cAdvisor uses no-new-privileges
  • SSM Session Manager: Management access without SSH keys or bastion hosts

Threat Analysis

Telemetry Data Poisoning

Vector: Attacker sends fake telemetry to /collect/*

Mitigations: CORS restricted to app domain, WAF rate limiting (2000 req/5min/IP), max payload 4MiB, WAF OWASP Core Rules, Known Bad Inputs ruleset

Residual Risk: Determined attacker can replay from allowed origin (low impact — polluted dashboards, not data breach)

Grafana Unauthorized Access

Vector: Attacker tries to reach Grafana dashboards

Mitigations: No ALB listener rule for /grafana/*, no SG rule from ALB to port 3001, SSM-only access requires AWS IAM credentials

Residual Risk: None — three independent layers prevent access (ALB routing, SG, SSM auth)

Container Escape / Privilege Escalation

Vector: Compromised container attempts host access

Mitigations: Non-root containers (UID 10001), no privileged mode, read-only config mounts, Docker bridge network isolation, no-new-privileges on cAdvisor

Residual Risk: cAdvisor has device access (/dev/kmsg) for kernel metrics

Credential Exposure

Vector: Secrets leaked in code or config

Mitigations: .env in .gitignore (never tracked), Terraform sensitive = true, no default passwords in docker-compose, SSM parameter injection

Residual Risk: None in committed code — credentials exist only in runtime environment

S3 Data Exfiltration

Vector: Attacker accesses observability data in S3

Mitigations: S3 public access blocked, IAM scoped to single bucket, AES-256 encryption at rest, VPC-internal access only, 7-day lifecycle deletion

Residual Risk: Compromised EC2 instance profile could read/write bucket (standard cloud risk, no path from internet)

Defense-in-Depth Layers

Layer Component Protection
1. Edge AWS WAF Rate limiting (2000 req/5min/IP), OWASP Core Rules, Known Bad Inputs blocking
2. Transport ALB + HTTPS TLS encryption, path-based routing, no Grafana routes exposed
3. Network Security Groups Source-restricted ingress (ALB SG for Faro, ECS SG for OTLP/Pyroscope)
4. Application CORS + Alloy Origin restriction to app domain, payload size limits (4MiB max)
5. Container Docker Non-root (UID 10001), no privileged mode, read-only config mounts, no-new-privileges
6. Access SSM + IAM Grafana requires AWS IAM credentials for SSM tunnel; no SSH, no bastion
7. Data S3 + KMS AES-256 encryption at rest, public access blocked, 7-day lifecycle deletion

Cost Analysis

All observability resources are gated behind the enable_observability feature flag. When disabled, the additional cost is zero.

Resource Monthly Cost Conditional
EC2 t3.medium ~$30 enable_observability
S3 storage <$1 (dev) enable_observability
ALB rules $0 (reuses existing) enable_observability
Total enabled ~$31
Total disabled $0

The EC2 instance follows the same ephemeral lifecycle as other runtime resources—destroyed during off-hours and recreated on demand. S3 data in the base workspace persists across these cycles.

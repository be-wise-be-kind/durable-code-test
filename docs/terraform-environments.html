Environments

Comparison of dev, staging, and production environment configurations

Overview

Each environment is configured via a .tfvars file in infra/environments/. These files control resource sizing, cost optimization features, and security posture. All environments deploy to us-west-2.

Dev Development

Domain: dev.durableaicoding.net

Strategy: Aggressive cost optimization with Fargate Spot and auto-shutdown.

Budget: $10/month

Staging Staging

Domain: staging.durableaicoding.net

Strategy: Balance of cost efficiency and production-like behavior.

Budget: $15/month

Prod Production

Domain: durableaicoding.net

Strategy: Reliability and security focus, 24/7 availability.

Budget: $25/month

Backend Configuration

Each workspace and environment has a dedicated backend configuration file in infra/terraform/backend-config/. The state path follows the pattern {workspace}/{env}/terraform.tfstate.

Environment Workspace Backend Config File State Path
Dev Base base-dev.hcl base/dev/terraform.tfstate
Dev Runtime runtime-dev.hcl runtime/dev/terraform.tfstate
Staging Base base-staging.hcl base/staging/terraform.tfstate
Staging Runtime runtime-staging.hcl runtime/staging/terraform.tfstate
Prod Base base-prod.hcl base/prod/terraform.tfstate
Prod Runtime runtime-prod.hcl runtime/prod/terraform.tfstate

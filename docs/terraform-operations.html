Operations

Just commands, CI/CD workflows, scripts reference, and common operational procedures

Just Commands Reference

All infrastructure operations use the just infra command. Scope options: base, runtime (default), all, bootstrap.

Command Arguments Description
just infra check-aws — Verify AWS credentials and display caller identity
just infra init SCOPE Initialize Terraform workspace with backend config
just infra plan SCOPE Plan infrastructure changes (dry run)
just infra up SCOPE AUTO Deploy infrastructure (AUTO: true for auto-approve)
just infra down SCOPE Destroy infrastructure (prompts for confirmation)
just infra down-force SCOPE Force destroy (no refresh, auto-approve)
just infra status — Show infrastructure status for base and runtime
just infra state SCOPE List Terraform state resources
just infra output SCOPE FORMAT Show Terraform outputs (FORMAT: text or json)
just infra fmt — Format all Terraform files recursively
just infra validate SCOPE Validate Terraform configuration
just infra unlock SCOPE LOCK_ID Force unlock Terraform state (5-second safety delay)
Usage Examples ▼ 
# Check AWS credentials first
just infra check-aws

# Initialize and plan runtime
just infra init runtime
just infra plan runtime

# Deploy runtime with auto-approve
just infra up runtime true

# Deploy everything (base then runtime)
just infra up all

# Tear down runtime only
just infra down runtime

# Check what's deployed
just infra status

# View runtime outputs as JSON
just infra output runtime json

CI/CD Workflows

GitHub Actions workflows in .github/workflows/ automate infrastructure and deployment operations.

up.yml Infrastructure Up

Schedule: Daily at 3 AM UTC (8 PM MST)

Manual: Dispatch with environment, scope, deploy-app, dry-run options

Initializes Terraform, applies infrastructure, waits for ALB readiness, optionally deploys the application, and verifies health.

down.yml Infrastructure Down

Schedule: Daily at 8 AM UTC (midnight PST)

Manual: Dispatch with environment, scope, dry-run options

Checks status, destroys runtime (and optionally base), calculates cost savings (~$2.10/day for runtime).

deploy.yml Application Deploy

Trigger: After Infrastructure Up completes, or manual dispatch

Manual: Dispatch with environment selection (dev/staging/prod)

Builds Docker images, pushes to ECR, updates ECS task definitions, deploys services, waits for stability (10 min timeout).

terraform-test.yml Terraform Tests

Trigger: PRs/pushes affecting infra/**, justfile

Jobs: validate-terraform (matrix: base, runtime), validate-scripts, test-just-targets, integration-test

Runs format check, validation, tfsec scan, ShellCheck, and just target tests.

Daily Schedule

Time (UTC) Time (MST) Workflow Action
3:00 AM 8:00 PM up.yml Infrastructure up + app deploy
8:00 AM 12:00 AM (midnight) down.yml Infrastructure teardown

Scripts Reference

Shell scripts in infra/scripts/ implement the low-level Terraform operations.

Script Purpose
workspace-init.sh Initialize and select Terraform workspaces with backend config
workspace-deploy.sh Unified deployment orchestrator (dispatches to base then runtime)
workspace-deploy-base.sh Deploy persistent base infrastructure (VPC, ECR, Route53, ACM)
workspace-deploy-runtime.sh Deploy ephemeral runtime infrastructure (ECS, ALB, listeners)
workspace-destroy.sh Unified destruction orchestrator (destroys in proper order)
workspace-destroy-base.sh Destroy base infrastructure (VPC, ECR, Route53) — destructive
workspace-destroy-runtime.sh Destroy runtime infrastructure (ECS, ALB listeners)
workspace-status.sh Display status of base and runtime workspaces
deploy-app.sh Build Docker images and deploy to ECS (frontend + backend)
setup-terraform-backend.sh Bootstrap: create S3 state bucket and DynamoDB lock table
check-domain-availability.sh Check domain availability via Route53 or whois

Common Operations

Deploy New Code

# Option 1: Manual via just (requires running infra)
just infra check-aws
just infra status      # Verify infra is up

# Option 2: Trigger deploy workflow from GitHub Actions
# Go to Actions → "Deploy Application" → Run workflow

Bring Infrastructure Up

# Full stack: base + runtime + app deploy
just infra up all

# Or step by step:
just infra init base
just infra up base
just infra init runtime
just infra up runtime

Bring Infrastructure Down

# Destroy runtime only (preserves VPC, ECR, certs)
just infra down runtime

# Force destroy (no confirmation, no refresh)
just infra down-force runtime

Check Status

# Show what's deployed across both workspaces
just infra status

# View specific outputs
just infra output runtime
just infra output base json

# List resources in state
just infra state runtime
